## Critical Discovery: Two Design Systems

The site operates **two parallel design systems** simultaneously:

| Attribute | v1 (Legacy) | v2 (Current) |
|-----------|-------------|--------------|
| **Background** | Light/white | Dark `#020617` |
| **Brand name** | "Detect Hidden Fees" (with spaces) | "DetectHiddenFees" (no spaces) |
| **CSS** | External `style.css` | Inline `<style>` tags |
| **Navigation** | Present in some pages | Only logo link |
| **Accent color** | `#1a3a5c` (navy) | `#2563eb` (blue) |
| **Glow orbs** | None | Three floating orbs |
| **Pages affected** | ~25-35 legacy pages | ~175-185 current pages |

This split creates:
- Brand name inconsistency for AI entity recognition
- Duplicate content across both versions
- Different navigation experiences
- Conflicting design language

**All v1 pages must be either converted to v2 or consolidated/removed.**

## Orphan Page Risk

With **no navigation menu** in the HTML body (only logo link), **every page except the homepage** is technically an orphan. Users can only navigate via:
1. Footer links (present on most pages)
2. Inline body links to related content
3. Alphabetical listing page (`alphabet-links.html`)

This severely limits:
- User navigation between related pages
- Internal PageRank flow
- AI crawler discovery of related content
