## Scoring Methodology

Each opportunity is scored on four dimensions (1-5 scale):

| Score | User Intent | Conversion Potential | SEO Opportunity | Tool Relationship |
|-------|-------------|---------------------|:----------------:|:-----------------:|
| 5 | Direct question consumers ask AI | User ready to upload document | Low competition, high demand | Directly leads to tool usage |
| 4 | Common consumer search | User researching before action | Moderate competition | Supports existing tool |
| 3 | Related informational need | Indirect conversion path | Competitive but achievable | Related to tool ecosystem |
| 2 | Broad educational query | Low immediate conversion | High competition | Weak tool relationship |
| 1 | Niche or low-volume question | Minimal conversion path | Minimal opportunity | No tool relationship |

**Priority Score = (Intent × 0.30) + (Conversion × 0.30) + (SEO × 0.20) + (Tool × 0.20)**

## CONTENT GOVERNANCE RULES

### When To Create A New Page
Create a new page when:
1. The question has **no existing dedicated content** on the site
2. The question has **demonstrable search demand** (GSC data, AI answer frequency)
3. The question **leads to a tool conversion** (can point to an analysis page)
4. The content can be **800+ words of unique value** (not a thin FAQ entry)

### When To Add To An Existing Page
Add to an existing page when:
1. The question is **related but secondary** to the main page topic
2. The existing page **already ranks** for the broader topic
3. Adding an FAQ entry would satisfy the intent without creating thin content
4. The question supports the **main page's conversion goal**

### When To Redirect/Consolidate
Redirect or consolidate when:
1. **Multiple pages** compete for the **exact same search intent**
2. Content is **thin or outdated** and cannot support 800+ words
3. One page is **clearly more authoritative** (more backlinks, stronger schema, higher traffic)
4. The pages **cannibalize each other** in search results
